# Sumble Design Partner Proposal

## Why Sumble

Sumble is a strong candidate because:

- they have a live MCP and public setup docs across multiple AI tools
- MCP access is already part of their commercial product
- they are a good proxy for the official MCP vendor customer
- they appear to have budget and urgency

## Goal

Validate that Pluxx can become the authoring and distribution layer for official MCP vendors, not just an OSS scaffolder.

## What We Would Offer

- create or harden an official Pluxx-generated source project for Sumble MCP
- generate and validate outputs for core targets
- improve release discipline around plugin packaging and versioning
- reduce platform-specific drift in docs and configuration over time
- feed product learnings back into Pluxx Core and future hosted vendor features

## Scope

- import Sumble MCP into a maintained Pluxx source project
- produce platform outputs for at least Claude Code, Cursor, and Codex
- add a release checklist and verification flow
- validate auth and runtime behavior for their real MCP setup
- document where platform-specific setup still leaks through

## Non-Goals

- rebuilding Sumble’s MCP
- owning their hosted MCP infrastructure
- building custom enterprise governance features solely for this engagement

## Design Partner Ask

- `6-8 weeks`
- one product or engineering owner on their side
- `45 minutes` weekly
- fast feedback on real packaging, setup, and release friction
- permission to use anonymized learnings
- ideally logo or case-study rights later, but not required upfront

## Success Criteria

- Sumble has one maintained plugin source repo instead of fragmented platform handling
- core target outputs build and validate cleanly
- ongoing MCP changes can be synced without resetting curated work
- their team says Pluxx materially improves the official plugin story
- we identify the minimum viable hosted and vendor features worth charging for

## What We Learn

- where official MCP vendors actually feel pain
- whether the buyer cares more about packaging, release confidence, analytics, or distribution
- which future paid surfaces matter most

Candidate future paid surfaces:

- hosted publish pipeline
- release history
- compatibility verification
- analytics
- branded official plugin registry and distribution

## Commercial Structure Options

### Option A

- free design partner
- exchange: deep feedback and future case-study rights

### Option B

- discounted paid pilot
- `$3k - $7k`

### Option C

- concierge implementation
- credited toward a future annual vendor plan

## Recommendation

Lead with free or very low-cost design partner terms.

Optimize for:

- speed
- feedback quality
- product learning
- referenceability

Do not optimize for early margin.

## What Not To Pitch Yet

- publish to all marketplaces as the whole story
- generic analytics as the headline
- enterprise governance unless they ask for it
- built-in model orchestration
- MCP hosting

## Positioning Line

Pluxx helps MCP vendors turn one live MCP into a maintained, official multi-agent plugin distribution workflow.
