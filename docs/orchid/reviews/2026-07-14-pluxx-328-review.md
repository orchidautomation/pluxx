# PLUXX-328 Phase 5 Review

## Verdict

Phase 5 meets the frozen core-four acceptance contract. The compiler recovers identity manifests, registration and discovery topology, lifecycle entrypoints, helper payloads, host-native extensions, and source-only distribution evidence as adjuncts owned by the existing nine canonical buckets. It does not add a tenth bucket, a competing capability table, or a secondary-host surface.

The pinned Compound Engineering, Hyperframes, and Superpowers inventories contain 44 exact source rows. Their 12 Claude Code, Cursor, Codex, and OpenCode receipts contain 176 deterministic compiled rows bound to source and compiled plugin identity, version, revision, inventory digest, host, compiler output digest, installed receipt digest, install ownership, and evidence tier.

## Review Findings Incorporated

- P1: reserved receipt paths and pre-existing non-manifest targets could create competing ownership. Fixed with reserved-path refusal and fail-closed collision checks before publication.
- P1: runtime proof accepted ownership digest facts without binding their values to the receipt and ownership record. Fixed with exact SHA-256 fact cross-binding before ownership can count as proven.
- P1: the pinned inventory test did not independently verify the public source fixtures. Fixed with an exact-revision audit that rehashes all 44 source files and executable modes.
- P1: source symbolic-link components could escape the fixture root. Fixed by rejecting symbolic-link components before any source read.
- P1 (final independent review): a pre-existing target symbolic link could be read during manifest merging before the transaction layer refused it. Fixed by rejecting every target symbolic-link component before target existence or content checks, with a regression test.
- P1: the receipt omitted the compiled plugin identity and could conflate fixture provenance with generated artifact identity. Fixed with separate strict source and compiled identity bindings.
- P2: adjunct owner and host enums duplicated canonical registries. Fixed by deriving owner values from the canonical compiler buckets and core-four hosts from the existing install/discovery registry.
- P2: migration detection was embedded in the already-large migration command and obscured provenance. Fixed by extracting the detector and persisting the complete recovered inventory and evidence tier in the migration receipt.
- P2: migration accepted missing declared OpenCode entrypoints and did not cover the recovered topology. Fixed with fail-closed declared-entrypoint validation and expanded four-manifest, three-marketplace, loader, and provenance tests.
- P2: provenance and plugin-name schemas were permissive or duplicated. Fixed with strict nested provenance and one dependency-neutral compiler contract.
- P2: payload publication and receipt publication were separate transactions. Fixed by preparing exact outputs and publishing payloads plus `distribution/adjuncts.receipt.json` through one rollback-capable transaction.
- P2: identical pre-existing outputs could be treated as safely owned. Fixed by refusing all pre-existing non-manifest outputs regardless of byte equality.
- P2: inventory mutations could survive recomputed enclosing hashes. Fixed by recomputing the canonical inventory digest during receipt validation.
- P2: generated runtime proof did not require the exact adjunct receipt path and identity. Fixed by validating the canonical receipt path, compiled identity, exact ownership entry, and digest facts.
- P2 (final independent review): an executable source with `executable: false` could be silently published as non-executable. Fixed by requiring exact executable-bit parity in both directions, with a regression test.

No valid P0-P2 finding remains unresolved. Project-standards, security, API, reliability, agent-native, adversarial, and maintainability review findings were either incorporated above or resolved as non-findings because the canonical receipt remains a deterministic shared-workspace artifact rather than hidden process state.

## Evidence Coverage

- Source inventory: Compound Engineering 12, Hyperframes 15, Superpowers 17; 44 exact rows total.
- Compiled inventory: 176 rows across 12 fixture/host receipts.
- Outcomes: 44 translate, 112 degrade, 20 drop, 0 preserve for source-inspected external fixtures.
- Receipt identity: 12 unique fixture/host identities and 12 unique compiler output digests.
- Install evidence: 12 isolated `fake-home-install` ownership receipts; no live plugin state was read or changed.
- Host portfolio: Claude Code, Cursor, Codex, and OpenCode only.
- Phase 3 ceiling: discovery remains environment-unavailable in 12/12, activation unsupported in 12/12, behavior environment-unavailable in 12/12, and all 324 orchestration outcomes remain degraded.

## Residual Risk And Phase 6 Handoff

Generated registration artifacts are not real-host discovery or activation evidence. Phase 6 must make the exact-revision 44-file audit, 12-case receipt freshness and determinism, migration/generator compatibility derivation, and installed-runtime evidence boundary release-gating. It must also exercise both symlink and copied install-ownership preimages; the current deterministic portfolio proves the symlink-root path while copied-install preimage coverage remains the explicit release-gate residual. Secondary hosts remain out of scope.
