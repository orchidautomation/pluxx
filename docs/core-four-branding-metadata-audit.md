## Real Example Verification Notes

### Firecrawl (`example/firecrawl-plugin`)

- No `brand` block in config.
- Generated core-four outputs have no brand/interface metadata, consistent with current generators.

### Prospeo (`examples/prospeo-mcp`)

- Config sets `brand.displayName`, `shortDescription`, `category`.
- Codex output includes:
  - `.codex-plugin/plugin.json -> interface.displayName, interface.shortDescription, interface.category`
- Cursor and Claude do not emit these fields; OpenCode wrapper has no brand interface layer.

### Megamind (`example/megamind`)

- Config sets rich brand metadata: display/short/long/category/color/icon/defaultPrompts/websiteURL.
- Codex output includes rich interface metadata as expected.
- Cursor output includes only `homepage` and `logo`.
- Claude and OpenCode do not surface brand metadata.
- `privacyPolicyURL` / `termsOfServiceURL` appear in Codex output only because this example sets `platforms.codex.interface` overrides.

### Pluxx (`example/pluxx`)

- Config sets `privacyPolicyURL` and `termsOfServiceURL` under `brand`.
- Codex output does not include those fields.
- Confirms drift between modeled schema fields and actual generator mapping.

## Drift Findings

1. `src/schema.ts` models `brand.privacyPolicyURL` and `brand.termsOfServiceURL`, but `src/generators/codex/index.ts` does not map them from `brand` into `manifest.interface`.
2. Claude and OpenCode currently provide no generator-level brand/interface manifest layer despite `brand` being modeled as a core primitive.
3. Cursor brand support is intentionally narrow (`homepage`, `logo`) compared to Codex's rich interface block.
4. Existing docs can be read as if brand support is broadly portable; in practice, rich support is Codex-centric today.

## Recommendations

1. Keep `brand` as a core primitive, but document it as target-graded rather than uniformly portable.
2. Decide and codify one of these for policy URLs:
   - map `brand.privacyPolicyURL` and `brand.termsOfServiceURL` into Codex `interface` directly, or
   - remove/deprecate those fields from `BrandSchema` and require target overrides.
3. Keep `screenshots` and `defaultPrompts` positioned as Codex-first fields until other hosts expose equivalent file-backed surfaces.
4. Stop implying that Claude/OpenCode consume brand metadata beyond indirect copy text usage.

## Suggested Contract Language

- Core primitive: `brand`.
- Current effective host support:
  - Claude Code: none (manifest-level brand metadata not emitted)
  - Cursor: minimal (`homepage`, `logo`)
  - Codex: rich `interface` metadata
  - OpenCode: none (no dedicated brand surface in generated wrapper/package)
