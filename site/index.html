One config to rule them all

Define your plugin in a single TypeScript config. plugahh generates the correct output for every platform.

// plugahh.config.ts import { definePlugin } from 'plugahh' export default definePlugin({ name: 'my-plugin', description: 'What your plugin does', author: { name: 'Your Name' }, mcp: { 'my-server': { url: 'https://api.example.com/mcp', auth: { type: 'bearer', envVar: 'API_KEY' }, }, }, brand: { displayName: 'My Plugin', color: '#6366f1', }, targets: ['claude-code', 'cursor', 'codex', 'opencode'], })

MCP auth translated automatically

if (!process.env.API_KEY) client.app.log({ level: "warn" })

How it works

Four commands. Zero lock-in.

npx plugahh init
Scaffold your plugin config with an interactive setup wizard.
Add your content
Write skills, configure MCP servers, add hooks and brand metadata.
npx plugahh build
Generate native plugin packages for every target platform in parallel.
npx plugahh install
Symlink to your local tools for instant testing. Then ship it.
