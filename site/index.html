One config to rule them all

Define your plugin in a single TypeScript config. pluxx generates the correct output for every platform.

// pluxx.config.ts import { definePlugin } from 'pluxx' export default definePlugin({ name: 'my-plugin', description: 'What your plugin does', author: { name: 'Your Name' }, mcp: { 'my-server': { url: 'https://api.example.com/mcp', auth: { type: 'bearer', envVar: 'API_KEY' }, }, }, brand: { displayName: 'My Plugin', color: '#6366f1', }, targets: ['claude-code', 'cursor', 'codex', 'opencode'], })

How it works

Four commands. Zero lock-in.

init
Scaffold
Interactive setup creates your pluxx.config.ts with MCP, hooks, and targets.
write
Add content
Write skills, configure MCP servers, add hooks and brand metadata.
build
Generate
Native plugin packages for every target platform, generated in parallel.
install
Ship
Symlink to local tools for testing, then publish to marketplaces.
