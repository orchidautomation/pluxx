Import once. Compile for the core four.

Pluxx can start from a raw MCP or an existing host-native plugin, normalize the durable source project, validate it, and produce native bundles for each primary target.

// pluxx.config.ts import { definePlugin } from '@orchid-labs/pluxx' export default definePlugin({ name: 'my-plugin', description: 'What your plugin does', author: { name: 'Your Name' }, mcp: { 'my-server': { url: 'https://api.example.com/mcp', auth: { type: 'header', envVar: 'API_KEY', headerName: 'X-API-Key', headerTemplate: '${value}', }, }, }, brand: { displayName: 'My Plugin', color: '#6366f1', }, targets: ['claude-code', 'cursor', 'codex', 'opencode'], })

Runtime intent translated automatically

How it works

Import, validate, refine only where models help, then ship native bundles.

init
Import your source
Start from a raw HTTP, SSE, or stdio MCP, or migrate an existing host-native plugin into a valid Pluxx project.
doctor
Validate the baseline
Run doctor and test so you know the deterministic scaffold is healthy before any semantic refinement.
agent
Refine with a host agent
Agent Mode prepares context and prompts so Claude, Cursor, Codex, or OpenCode improve only the managed refinement surfaces.
build
Compile and ship
Build native platform bundles, install locally, and keep the source project updated later with pluxx sync --from-mcp.
