# Autopilot

`pluxx autopilot` is the production command for running import, agent refinement, and verification in one flow.

## Command

```bash
pluxx autopilot --from-mcp <source> --runner <codex|claude|cursor|opencode>
```

Example:

```bash
pluxx autopilot \
  --from-mcp https://mcp.playkit.sh/mcp \
  --runner codex \
  --mode standard \
  --yes \
  --name playkit \
  --display-name "PlayKit" \
  --targets claude-code,cursor,codex,opencode \
  --grouping workflow \
  --hooks safe \
  --auth-env PLAYKIT_API_KEY \
  --auth-type header \
  --auth-header X-API-Key \
  --auth-template '${value}'
```

## What It Runs

Autopilot orchestrates existing CLI capabilities:

1. MCP import and deterministic scaffold generation
2. optional taxonomy pass
3. optional instructions pass
4. optional review pass
5. optional final verification (`pluxx test`)

Pass selection depends on `--mode`, MCP quality signals, and whether context inputs were provided.

## Modes

- `quick`: minimum agent work
- `standard`: balanced defaults (review is opt-in with `--review`)
- `thorough`: taxonomy + instructions + review by default

## Context Inputs

Provide context to improve semantic passes:

- `--website <url>` product website
- `--docs <url>` public docs
- `--context <files...>` additional local files

## Verification and Logs

- default behavior includes final verification (`pluxx test`)
- `--no-verify` skips the verification stage
- `--verbose-runner` streams full headless runner stdout/stderr

## Runtime/Auth Notes

Autopilot supports the same auth flags as `init --from-mcp`:

- `--auth-env`
- `--auth-type bearer|header|platform`
- `--auth-header`
- `--auth-template`
- `--runtime-auth inline|platform`
- `--oauth-wrapper`

For OAuth-first MCPs, use `--oauth-wrapper` when direct introspection cannot complete without an interactive handshake.

## Dry Run

Use dry run to inspect the full plan and command workload:

```bash
pluxx autopilot --from-mcp https://example.com/mcp --runner codex --mode standard --yes --dry-run
```

Use JSON output for automation:

```bash
pluxx autopilot --from-mcp https://example.com/mcp --runner codex --yes --json
```
