# Getting Started With Pluxx

This guide walks through the production baseline for a new Pluxx project:

1. import from an MCP source
2. verify scaffold health
3. build and install host bundles
4. re-sync when the MCP changes

## Prerequisites

- Node.js 18+
- Bun 1.0+
- `npx @orchid-labs/pluxx` available

## 1. Import From MCP

Remote HTTP MCP:

```bash
npx @orchid-labs/pluxx init --from-mcp https://example.com/mcp --yes
```

Local stdio MCP:

```bash
npx @orchid-labs/pluxx init --from-mcp "npx -y @acme/mcp" --yes
```

Legacy SSE MCP:

```bash
npx @orchid-labs/pluxx init --from-mcp https://example.com/sse --transport sse --yes
```

Authenticated HTTP MCP (custom header):

```bash
npx @orchid-labs/pluxx init \
  --from-mcp https://mcp.playkit.sh/mcp \
  --yes \
  --auth-env PLAYKIT_API_KEY \
  --auth-type header \
  --auth-header X-API-Key \
  --auth-template '${value}'
```

## 2. Review The Scaffold

```bash
npx @orchid-labs/pluxx doctor
npx @orchid-labs/pluxx lint
npx @orchid-labs/pluxx eval
```

Generated project shape:

```text
my-plugin/
  pluxx.config.ts
  INSTRUCTIONS.md
  skills/
  .pluxx/mcp.json
  dist/
```

`INSTRUCTIONS.md` and `SKILL.md` files contain generated and custom sections. Keep your manual edits in custom sections so `sync` can safely refresh generated content.

## 3. Build, Install, Test

```bash
npx @orchid-labs/pluxx build
npx @orchid-labs/pluxx install --target claude-code
npx @orchid-labs/pluxx test
```

Helpful previews:

```bash
npx @orchid-labs/pluxx build --dry-run
npx @orchid-labs/pluxx install --dry-run
```

If your plugin defines hook commands, install prompts for trust confirmation unless you pass `--trust`.

## 4. Re-Sync Later

When MCP tools or descriptions change, refresh the scaffold:

```bash
npx @orchid-labs/pluxx sync
```

If your MCP endpoint changed:

```bash
npx @orchid-labs/pluxx sync --from-mcp https://mcp.example.com/mcp
```

## 5. One-Command Flow (Autopilot)

```bash
npx @orchid-labs/pluxx autopilot \
  --from-mcp https://example.com/mcp \
  --runner codex \
  --mode standard \
  --yes
```

Use autopilot when you want import + agent refinement + verification in one command. See [Autopilot](/workflows/autopilot) for full behavior.
