---
title: Core vs Deferred Primitives
---

## Core primitives

- `skills`
- `instructions`
- `mcp`
- `userConfig`
- `commands`
- `agents`
- `hooks`
- `permissions`
- `brand`
- `assets/scripts`
- `taxonomy`

## Deferred primitives

- `outputStyles`
- `lspServers`
- `bin/`
- `monitors`
- `channels`
- `apps` abstraction
- plugin data-dir abstraction
- statuslines
- themes / keybindings
- sandbox policy

These boundaries keep Pluxx focused on newcomer-critical plugin authoring.
