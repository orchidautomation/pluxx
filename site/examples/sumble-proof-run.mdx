## What We Ran

Deterministic scaffold:

```bash
SUMBLE_API_KEY=... \
bun bin/pluxx.js init \
  --from-mcp https://mcp.sumble.com \
  --yes \
  --name sumble-plugin \
  --display-name "Sumble" \
  --author "Orchid Labs" \
  --targets claude-code,cursor,codex,opencode \
  --grouping workflow \
  --hooks safe \
  --auth-env SUMBLE_API_KEY \
  --auth-type bearer
```

```bash

```

## What The Warnings Actually Were

### 1. Install trust warning from `doctor`

This warning is about local shell hooks, not about Sumble or the import failing:

```text
WARNING hooks-trust-required [pluxx.config.ts] Hook commands require install trust
Fix: Review the commands carefully. Users will need to opt in with pluxx install --trust.
```

Plain English:

- Pluxx generated local shell hooks under `scripts/`
- that is useful, but users should explicitly trust them at install time
- Pluxx warns here on purpose instead of silently normalizing shell execution

### 2. Codex hook parity warning from `build`

This is the warning that matters most for platform honesty:

```text
WARN codex-hooks-external-config [Codex] pluxx.config.ts:
Codex plugin docs currently separate hook configuration from plugin packaging,
so Pluxx does not bundle Codex hooks into generated plugin output.
If you want Codex to run these hooks, configure them in ~/.codex/hooks.json
or <repo>/.codex/hooks.json and enable codex_hooks = true in Codex itself.
```

Plain English:

- Claude Code and Cursor got packaged hook output
- Codex did not get the same packaged hook parity
- the plugin still built and tested for Codex
- the caveat is specifically about hooks, not the whole plugin

That warning should stay visible because pretending the platforms are identical would create more distrust later.

## Real Product Learning

This run exposed and fixed a real Pluxx bug.

The Codex runner could complete edits and then hang during session finalization when invoked as:

```text
codex exec
```

Pluxx now uses:

```text
codex exec --ephemeral
```

for the Codex headless worker path, which keeps agent and autopilot runs more stable.
