title: Codex
---

Use this target for CLI-first workflows and findings-first review loops.

## Build and Install

```bash
npx @orchid-labs/pluxx build --target codex
npx @orchid-labs/pluxx install --target codex
```

Local install path:

- `~/.codex/plugins/<plugin-name>`

Pluxx also updates local marketplace metadata at:

- `~/.agents/plugins/marketplace.json`

## Runtime Files

Key generated files under `dist/codex/`:

- `.codex-plugin/plugin.json`
- `.mcp.json`
- `AGENTS.md`
- `.codex/hooks.json` (when hooks are configured)
- `.codex/permissions.generated.json`

## Permissions Caveat

Codex permission enforcement is external to the plugin bundle. Pluxx emits `.codex/permissions.generated.json` as a canonical mirror so you can wire it into Codex policy and hooks.

## Invocation Model

Codex relies on `@plugin` and skill routing rather than slash-command parity. Keep workflow-critical semantics in skills and instructions.
