title: Claude Code
---

Use this target when Claude Code is the primary interactive host for your plugin.

## Build and Install

```bash
npx @orchid-labs/pluxx build --target claude-code
npx @orchid-labs/pluxx install --target claude-code
```

Pluxx registers a local Claude marketplace entry and installs the plugin through Claude's plugin tooling.

If Claude Code is already running, execute `/reload-plugins` in the active session.

## Local Install Location

Installed plugin data is managed under your home directory through Claude marketplace registration.

Relevant generated files inside `dist/claude-code/` include:

- `.claude-plugin/plugin.json`
- `.mcp.json`
- `CLAUDE.md`
- `hooks/hooks.json` (when hooks are configured)

## Runtime Auth Behavior

- For `runtimeAuth: inline`, Pluxx can materialize auth headers/env in installed MCP config.
- For `runtimeAuth: platform`, auth is delegated to platform-managed flows.

## Command/Skill Name Collisions

When the same name exists in both `commands/<name>.md` and `skills/<name>/SKILL.md`, Pluxx keeps command-first UX and rewrites the colliding skill to a hidden internal shape so semantic routing still works.
