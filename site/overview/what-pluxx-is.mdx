title: What Pluxx Is
---

Pluxx keeps one maintained plugin source project for Claude Code, Cursor, Codex, and OpenCode.

Start from a live MCP when you have one, or author the source project directly. Then compile the native bundle each host expects.

## Source of truth

Pluxx keeps one editable source project and compiles per-host outputs.

```text
source project (you edit)
  - pluxx.config.ts
  - INSTRUCTIONS.md
  - skills/
  - scripts/
  - .pluxx/mcp.json

compiled bundles (generated)
  - dist/claude-code
  - dist/cursor
  - dist/codex
  - dist/opencode
```

## What is already real

Pluxx is no longer just a thin bundle generator.

- `pluxx eval` checks scaffold and prompt-pack quality, not just whether the project builds
- `pluxx migrate <path>` imports an existing host-native plugin into a Pluxx source project
- `pluxx doctor --consumer <bundle>` inspects built or installed plugin bundles from the end-user side
- `pluxx mcp proxy --record` / `--replay` gives you deterministic MCP tapes for debugging and CI

## Product flow

1. Start from an MCP or initialize a source project.
2. Run `doctor`, `lint`, `eval`, and `test` to validate the baseline.
3. Refine taxonomy and instructions with Agent Mode or autopilot when you need semantic cleanup.
4. Build native bundles and install them into your target hosts.
5. Sync later when the MCP changes.

## Quick start

```bash
npx @orchid-labs/pluxx init --from-mcp https://example.com/mcp
npx @orchid-labs/pluxx doctor
npx @orchid-labs/pluxx lint
npx @orchid-labs/pluxx build
npx @orchid-labs/pluxx test
```
