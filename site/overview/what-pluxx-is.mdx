---
title: What Pluxx Is
---

Pluxx turns one MCP or source project into native plugin bundles for Claude Code, Cursor, Codex, and OpenCode.

Keep one maintained source project, then compile the native bundle each host expects.

## Source of truth

Pluxx keeps one editable source project and compiles per-host outputs.

```text
source project (you edit)
  - pluxx.config.ts
  - INSTRUCTIONS.md
  - skills/
  - scripts/
  - .pluxx/mcp.json

compiled bundles (generated)
  - dist/claude-code
  - dist/cursor
  - dist/codex
  - dist/opencode
```

## Core commands

Pluxx includes more than scaffold generation:

- `pluxx eval` checks scaffold and prompt-pack quality before publish
- `pluxx migrate <path>` imports an existing host-native plugin into a Pluxx source project
- `pluxx doctor --consumer <bundle>` inspects a built or installed plugin bundle from the user side
- `pluxx mcp proxy --record` and `--replay` capture deterministic MCP sessions for debugging and CI

## Product flow

1. Start from an MCP or initialize a source project.
2. Run `doctor`, `lint`, `eval`, and `test` to validate the baseline.
3. Refine taxonomy and instructions with Agent Mode or autopilot when you need semantic cleanup.
4. Build native bundles and install them into your target hosts.
5. Sync later when the MCP changes.

## Quick start

```bash
npx @orchid-labs/pluxx init --from-mcp https://example.com/mcp
npx @orchid-labs/pluxx doctor
npx @orchid-labs/pluxx lint
npx @orchid-labs/pluxx build
npx @orchid-labs/pluxx test
```
