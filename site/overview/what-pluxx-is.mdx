---
title: What Pluxx Is
---

Pluxx is a plugin authoring and maintenance layer for MCP teams.

You keep one source project (`pluxx.config.ts`, skills, instructions, hooks, MCP wiring), and Pluxx compiles host-native bundles for Claude Code, Cursor, Codex, and OpenCode.

## Product Model

Pluxx is not an MCP server runtime. It is the authoring system between your MCP and host-specific plugin contracts.

- You own MCP behavior and deployment.
- Pluxx owns scaffold generation, validation, compilation, install helpers, and sync.

## Source-of-Truth Layout

```text
my-plugin/
  pluxx.config.ts
  INSTRUCTIONS.md
  skills/
    search/SKILL.md
  scripts/
    check-env.sh
  .pluxx/
    mcp.json
  dist/
    claude-code/
    cursor/
    codex/
    opencode/
```

- Edit source files in the project root.
- Treat `dist/*` as generated output.

## Core Workflow

```bash
npx @orchid-labs/pluxx init --from-mcp https://example.com/mcp --yes
npx @orchid-labs/pluxx doctor
npx @orchid-labs/pluxx lint
npx @orchid-labs/pluxx eval
npx @orchid-labs/pluxx build
npx @orchid-labs/pluxx test
npx @orchid-labs/pluxx install --target claude-code
```

Typical `build` output shape:

```text
Done! Output in dist/
  dist/claude-code/
  dist/cursor/
  dist/codex/
  dist/opencode/
```

## Runtime Notes

- Package: `@orchid-labs/pluxx`
- Preferred invocation: `npx @orchid-labs/pluxx ...`
- Runtime requirement: Bun available on `PATH`
- Core target focus: Claude Code, Cursor, Codex, OpenCode
