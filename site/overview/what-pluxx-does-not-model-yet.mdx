---
title: What Pluxx Does Not Model Yet
---

Pluxx intentionally keeps scope tight around core plugin-authoring primitives.

Out of current core scope:

- `outputStyles`
- `lspServers`
- `bin/` executables
- `monitors`
- `channels`
- `apps` abstraction
- plugin data-dir abstraction
- statuslines
- themes / keybindings
- sandbox and other runtime policy

These are documented for roadmap decisions, but they are not part of the current core model.

See [Core vs deferred primitives](/roadmap-boundaries/core-vs-deferred-primitives).
