# Demo Rewrites

These files are read-only proof artifacts for the flagship `docs-ops` example.

They show how a public Docsalot MCP surface can be:

- inspected
- pulled into context
- rewritten with stronger product and docs guidance

without requiring a publish step.

They intentionally sit on the read-only side of the flagship proof stack.

The separate authenticated write/publish lane now lives in:

- `../ORCHID-AUTHENTICATED-PUBLISH-PATH.md`

Current artifact set:

- `orchid-components-accordion.before.md`
- `orchid-components-accordion.after.md`

These files are not canonical Orchid source files.

They are Pluxx proof assets that demonstrate the rewrite workflow on a real public docs surface.
