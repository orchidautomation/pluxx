# People Research Patterns

When the user wants people, search for proof before profile pages.

Prioritize:

- writing
- talks
- repos
- launch posts
- technical or operator commentary

Good query shapes:

- `category:people staff engineer distributed systems observability writing`
- `category:people frontend engineer startup product design opinions`
- `category:people founder workflow automation enterprise go-to-market`

If the user wants a list:

- decide what qualifies first
- make the qualification criteria explicit in the output
