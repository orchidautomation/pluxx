# Platform Change Ops

Platform Change Ops is the maximal enterprise reference plugin for Pluxx.

It exists to prove that one maintained source project can carry:

- multiple remote MCP servers
- one bundled local stdio runtime
- runtime readiness
- explicit permissions
- risky-action hooks
- delegated specialist agents
- install and distribution metadata
- honest preserve / translate / degrade behavior across Claude Code, Cursor, Codex, and OpenCode

## What It Models

This example is shaped like an internal AI platform / DevEx operator pack:

- change intake
- blast-radius analysis
- release-risk review
- rollout comms
- change-window gating
- rollback planning
- installed-state verification

## Why This Example Exists

Current examples each prove part of the story:

- `docs-ops` proves workflow depth and a separated read/write docs lane
- `exa-plugin` proves agents, branding, and research orchestration
- `sumble-plugin` proves matcher-based mutation hooks
- `prospeo-mcp` proves local stdio MCP packaging

This example combines those surfaces in one place so the remaining translation and lifecycle gaps stop hiding in isolated fixtures.
